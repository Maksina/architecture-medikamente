# Разработка cutover-плана

## Выбор стратегии

Выбран подход **Parallel run**

**Причины:**
- *Нельзя отключать текущие системы.* 1С:Бухгалтерия и 1С:Торговля и склад, ККМ и файловый сервер обеспечивают ежедневную работу. Мгновенное переключение на новую платформу несет повышенные риски для учета и оплаты
- *Поэтапный ввод новых возможностей.* Сначала вводится портал записи и CRM при сохранении Excel/1С для остального; затем подключаются платёжный шлюз и интеграция с лабораторией; затем аналитический слой (CDC, Kafka, AirFlow, ClickHouse). Parallel run позволяет включать новые блоки по одному и сверять результаты со старыми.
- *Ограничения подхода и их учёт.* Двойное ведение (часть данных в Excel, часть в CRM) и риск рассинхронизации компенсируются: чёткими правилами «кто и что ведёт в какой системе», регулярной сверкой, ограниченным периодом параллельной работы по каждому процессу и использованием плана миграции данных, тестового контура, плана отката.

Комментарий: High-level design используется на этапе проектирования, сам переход использует Parallel run, а Branch by abstraction можно применять внутри этапов, например, при замене источника данных «записи на приём» с Excel на CRM.
