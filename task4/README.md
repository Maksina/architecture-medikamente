# Рекомендации по устранению узких мест

## Инфраструктура

*Узкие места: один сервер — единая точка отказа; ограниченная пропускная способность сети; нехватка CPU/RAM при росте нагрузки.*

| Мера | Приоритет |
|------|-----------|
| Провести оценку ресурсов (CPU, RAM, диск, сеть) под целевую нагрузку и план масштабирования (доп. серверы, виртуализация, облако) | высокий |
| Выделить отдельные контуры для продакшена и тестирования миграции | высокий |
| Внедрить мониторинг ресурсов (CPU, память, диск, сеть) и алерты при достижении порогов | высокий |
| Спланировать резервное копирование и восстановление для новых БД и критичных сервисов | средний |
| Рассмотреть сегментацию сети для изоляции критичных систем | средний |

## Программное обеспечение

*Узкие места: файловый режим 1С; несовместимость версий 1С при обновлениях; тяжёлые ETL/CDC конкурируют с операционными запросами.*

| Мера | Приоритет |
|------|-----------|
| Настроить пулы соединений к БД, лимиты запросов и таймауты в приложениях и API Gateway | высокий |
| Провести нагрузочное тестирование веб-интерфейсов, CRM и API до перехода на новую систему | высокий |
| Оценить переход 1С на серверный режим (или выделенный сервер БД) для снижения блокировок и улучшения резервного копирования | средний |
| Вынести тяжёлые ETL/CDC в отдельные окна или приоритизировать по ресурсам, чтобы не конкурировать с операционными запросами | средний |
| Документировать версии 1С, драйверов ККМ и зависимостей; тестировать обновления в тестовом контуре | низкий |

## Интеграция

*Узкие места: синхронизация старой и новой системы; API: задержки, недоступность, объём данных; OLE 1С–1С ненадёжный.*

| Мера | Приоритет |
|------|-----------|
| Спроектировать надёжную передачу данных ККМ → 1С/платёжный шлюз (ретраи, идемпотентность, мониторинг) | высокий |
| Определить контракт и SLA с лабораторией по API (доступность, лимиты, формат); реализовать ретраи и очередь для загрузки результатов | высокий |
| Ввести мониторинг: задержки, ошибки, объём очередей | высокий |
| В переходный период вести чёткие правила синхронизации старых и новых систем и периодическую сверку | средний |


## Персонал

*Узкие места: малая команда при росте количества сервисов; нехватка компетенций (k8s, Kafka); сопротивление пользователей.*

| Мера | Приоритет |
|------|-----------|
| Организовать обучение пользователей (ресепшен, кассиры, врачи) до перехода на веб-интерфейсы и CRM; подготовить инструкции и ответы на частые вопросы | высокий |
| Закрепить роли по миграции и поддержке (владелец данных, ответственный за 1С/ККМ, за новую платформу); при необходимости привлечь внешних специалистов | высокий |
| Формализовать регламенты эксплуатации (мониторинг, эскалация, резервное копирование) | средний |
| Спланировать развитие компетенций (Kubernetes, Kafka, безопасность) для команды | средний |

## Процессы

*Узкие места: нет плана миграции данных и отката; недостаточное тестирование; отсутствие регламентов, выстроенной поддержки.*

| Мера | Приоритет |
|------|-----------|
| Разработать пошаговый план миграции данных и провести пилотную миграцию на копиях | высокий |
| Ввести обязательное нагрузочное и функциональное тестирование перед переключением критичных процессов на новую систему | высокий |
| Подготовить план отката и провести учения по откату | высокий |
| Использовать тестовый контур для отладки и тестирования изменений | средний |
| Регулярно проводить сверку данных между старой и новой системой в переходный период | средний |

## Данные

*Узкие места: большой объём неструктурированных данных; низкое качество (дубли, форматирование); конфликты при параллельной работе старой и новой системы.*

| Мера | Приоритет |
|------|-----------|
| Провести инвентаризацию и оценку качества данных в Excel и файлах (дубли, форматы, ключи) до миграции; при необходимости очистка и нормализация | высокий |
| Спроектировать миграцию файлов с проверкой целостности и отображением в новой системе | высокий |
| Заложить запас по производительности под пятикратный рост объёма данных | средний |
| Разделить зоны и политики доступа к тегированным данным в аналитическом слое| средний |
