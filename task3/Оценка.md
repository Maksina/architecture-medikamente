# Оценка Data Encryption at Rest and In Transit

| Данные | В чём необходимость защиты | Инструменты защиты |
|---|---|---|
| ФИО, дата рождения, контакты (телефон, email) | 152-ФЗ: конфиденциальность ПДн; идентификация субъекта; риски утечки и мошенничества | At rest: TDE PostgreSQL, KMS (Vault). In transit: TLS 1.2+. Контроль: RBAC/ABAC (IAM), маскирование в логах/отчётах, аудит (ELK). |
| Адрес, место работы/учёбы, хронические заболевания | Расширенные ПДн и мед. сведения; 152-ФЗ, медицинская тайна (323-ФЗ) | At rest: шифрование в CRMDB и хранилище документов; теги PII_EXTENDED/MEDICAL. In transit: TLS; минимизация в API. ABAC, аудит, DLP. |
| Диагнозы, заключения, результаты анализов, медицинские карты | Медицинская тайна (323-ФЗ); репутационные и судебные риски | At rest: шифрование CRMDB, S3, KMS; тег MEDICAL. In transit: TLS для порталов, API лаборатории. Строгий RBAC, полный аудит, проверки в CI/CD. |
| Платёжные данные (суммы, привязка к пациенту, кассовые операции) | 152-ФЗ; финансовая и налоговая отчётность; риски мошенничества | At rest: шифрование 1С и CRMDB; KMS. In transit: TLS/VPN ККМ–1С, платёжный шлюз. Аудит, маскирование в отчётах. |
| Журналы записей на приём | ПДн и связка с мед. контекстом; минимизация по 152-ФЗ | At rest: шифрование в CRMDB; тег PII; разграничение (ресепшен/врач — только свои). TLS. RBAC, аудит. |
| Кадровые и зарплатные данные сотрудников | Служебные ПДн по 152-ФЗ; внутренняя политика | At rest: шифрование в 1С. In transit: TLS. RBAC, аудит. |
| Данные в clickhouse| Требования к обезличенным данным | At rest: шифрование; KMS. In transit: TLS. Обезличивание перед загрузкой; доступ только к обезличенным данным; аудит выгрузок. |
| Данные при интеграции с лабораторией| Медицинская тайна и ПДн при передаче третьей стороне | In transit: TLS; API Gateway с аутентификацией; минимизация полей в API. Логирование вызовов, теги MEDICAL. |
| Реквизиты поставщиков (1С Торговля и склад) | Служебные/коммерческие данные | At rest: шифрование БД 1С. In transit: TLS. RBAC, аудит при необходимости. |
| Справочники (типы услуг, номенклатура ТМЦ) | Целостность и доступность; низкая конфиденциальность | Резервное копирование; при необходимости шифрование. TLS в общих каналах. Стандартный доступ по ролям. |
